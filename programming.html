学習について

        
高校新課程の「情報」に対する共通テスト対策

        2025年の共通テストから「情報」が一般科目として加えられます。
        主な試験の項目は２つです。
        ひとつめが情報社会に関する知識全般。そしてもうひとつがプログラミングにおけるアルゴリズムの問題です。

        ＜１＞情報社会に関する知識の学習方法
        １．実技を伴う学習
        実際にコンピュータを触ってメールやネット検索などを体験しながらどのような仕組みになっているのかを学びます。
        またワードのようなドキュメント文書の作成やエクセル・スプレッドシートのような表計算も実際に活用して学びます。
        また自分でWEBサイトを構築したり、そのデータをネット上にアップします。
        またデータベースの構築や運用、プログラミングによる問題解決もします。
        この学習方法の弱点は適切な教材やカリキュラムでなければハードルが高くなりやすいということです。

        ２．座学による学習（実技を伴わない学習）
        教科書や問題集で学習します。
        この学習方法の弱点は実感が湧きにくいことです。

        ３．実技と座学の併用による学習
        実際の指導要領などで文部科学省がすすめている方針が実技と座学の並行学習です。
        理想的ですが適切な教員がいなければ実技部分が形だけになりやすいという弱点があります。

        ４．アルゴリズムを別のフレームワークで学習する
        レゴを用いたり、ドローンを用いたりして学習します。
        長所としては実際に動くものを使うことで実感が湧きやすいというのがあります。
        またコンピュータを用いてプログラムを書くことに抵抗がある人、苦手な人にとって取り組みやすいことがあります。
        将来、工学部で機械を動かす、ロボットを制御するなどを学びたい人にとって向いている学習ができます。
        レベルが上がると世界大会などもあり、世界の優秀な頭脳と対戦するまで限界に挑戦することもできます。
        ワクワクしながらレベルを上げつつ学べる人にとっては大学入試のプログラミング関連問題は簡単すぎるでしょう。

        短所としては大学入試への対策としては余分に学習することが多くなります。
        直接プログラミングを学べる人であれば１日〜１週間もあれば大学入試問題のプログラミング関連問題は解けるようになります。
        それに対してフレームワークの使い方などの学習が多くなります。
        また入試問題とは異なる形式が多く、実際にコンピュータで操作してものを動かすレベルにまで到達しないと大学入試には対応できません。
        課外活動としての取り組みの部分が大きくなり、時間がかかります。
        入試のためというよりも楽しみながら時間をかけて学習することが目的となります。
        

        ＜２＞Pythonにおけるプログラミング（アルゴリズム）の学習方法
        １．はじめに
        キーボード入力や実行、出力は指導者が行います。
        １行または２行のサンプルプログラムを用いて基本構文を学びます。
        基本構文は大きく分ければ４つしかありません。
        細かく分けても使用する構文（文法）の数は２０以下です。

        ２．問題を解く（基本構文）
        １〜２行のサンプル問題での出力結果を答えます。
        口頭や紙に書く形式で答えます。
        実行、出力は指導者が行います。

        ３．問題を解く（基本構文の組み合わせ）
        基本構文を組み合わせて１０行以下のプログラムについて答えます。
        実行、出力は指導者が行います。

        ４．アルゴリズムを考える（基本構文）
        基本構文で解けるような問題が与えられます。
        問題の解き方を考えます。
        解き方を考えたら指導者がそれをコードに書いて実行します。

        ５．アルゴリズムを考える（基本構文の組み合わせ）
        基本構文の組み合わせで解けるような問題が与えられます。
        問題の解き方を考えて指導者に伝えます。
        コードは指導者が書いて実行します。

        ６．プログラムを書いてみる（基本構文）
        基本構文で解けるような問題が与えられます。
        問題の解き方を考えてそれを紙に書きます。
        そのとおりに指導者が書いて実行します。

        ７．プログラムを書いてみる（基本構文の組み合わせ）
        基本構文の組み合わせで解けるような問題が与えられます。
        問題の解き方を考えてそれを紙に書きます。
        そのとおりに指導者が書いて実行します。

        ※ここからは実際に学習者がコンピュータを用いてプログラムの学習をします。
        ※PCに実行環境を構築することはトラブルが起きやすく難しいケースが多いので指導者側が準備します。
        ※学習者は「コードを書くこと」「実行ボタンを押すこと」の２つの動作が増えます。

        ８．プログラムを書いて実行してみる（写して実行）
        完成された短いプログラムが与えられます。
        同じように入力し、実行します。
        出力結果が予想通りになったかどうかを確認します。

        ９．プログラムを書いてみる（基本構文の実践）
        １〜２行で書けるような基本構文で実行できる問題が与えられます。
        問題に対するコードを自分で書き、実行して出力します。
        正解が出力されない場合はどこが違うのかを探します。

        １０．プログラムを書く（基本構文の組み合わせ）
        基本構文を組み合わせれば解けるような問題が与えられます。
        正解の出力を目指します。

        １１．アルゴリズムを考えてプログラムを書く
        基本構文の組み合わせを利用し、なおかつ問題解決のためのアルゴリズムを考えます。
        よくある手法を学んでいきます。

        １２．人間には解けない問題をプログラムで解く（アルゴリズムの構築）
        基本構文を組み合わせて数学の問題などを解きます。
        計算量が多い問題で人の手によって求めるのが難しい問題を解きます。

        １３．さまざまな問題を解く
        見た目は大学入試数学のような問題を解きます。
        アルゴリズムによって計算量が異なることを考慮したりオーバーフローに気をつけたりする問題を扱います。
        
        
        
プログラミングについて

        OSは Windows と mac のどちらでも扱っています。Linux にも対応します。
        設定済みのLinuxノートPCを授業時に貸すことも可能です。
        高校生までで扱うプログラミング言語の中心は python です。ただし javascript を扱う場合には html と css を扱うことがあります。
        プログラミングを用いた数学的な取り組みをする場合は python を用いることが多いです。
        使用するエディタは Visual Studio Code です。
        意欲的な方には Android アプリ開発やそれに伴う言語 Kotlin を用いた授業もしています。
        他にも C言語 Java を扱うことがあります。
    
        複数のプログラミング言語を使用するのは難しいことはありません。
        大学入試までに必要なレベルのアルゴリズムにおいては、それぞれの細かい仕様を覚えるレベルではないからです。
        一般的には難しいと思われがちなC言語であってもシンプルに利用できます。
        むしろJavaなどのクラスやオブジェクトなどの概念が出てくる方が難しいかもしれません。
    
        エンジニアを目指すレベルではないことに注意する必要があります。
        基本的なアルゴリズムを用いて数学の問題も解きます。
        繰り返し、条件分岐、配列やリスト、この３つを利用できるようになればほとんどのプログラムが書けます。
