プログラミングの授業について

    OSは Windows と mac のどちらでも扱っています。Linux にも対応します。
    授業時に設定済みのLinuxノートPCを貸して授業することも可能です。
    高校生までで扱うプログラミング言語の中心は python です。ただし javascript を扱う場合には html と css を扱うことがあります。
    プログラミングを用いた数学的な取り組みをする場合は python を用いることが多いです。
    使用するエディタは Visual Studio Code です。
    意欲的な方には Android アプリ開発やそれに伴う言語 Kotlin を用いた授業もしています。
    他にも C言語 Java を扱うことがあります。

    複数のプログラミング言語を使用するのは難しいことはありません。
    大学入試までに必要なレベルのアルゴリズムをそれぞれの細かい仕様を覚えるレベルではないからです。
    一般的には難しいと思われがちなC言語であってもシンプルに利用できます。
    むしろJavaなどのクラスやオブジェクトなどの概念が出てくる方が難しいかもしれません。

    エンジニアを目指すレベルではないことに注意する必要があります。
    基本的なアルゴリズムを用いて数学の問題も解きます。
    繰り返し、条件分岐、配列やリスト、この３つを利用できるようになればほとんどのプログラムが書けます。

git

    git init	リポジトリを新規作成
    git clone	リポジトリをコピー
    git gc	リポジトリを最適化
    git pull	リモートリポジトリの変更点をローカルリポジトリにマージ
    git push	ローカルリポジトリの変更点をリモートリポジトリにマージ
    git add	コミット対象のファイルを登録
    git commit	変更されたファイルをコミット（ローカルリポジトリに変更内容を入れ込む）
    git reset	直前のコミットを取消
    git revert	特定のコミットを取消
    git tag	コミットにタグを付ける
    git log	コミット履歴を表示
    git status	作業ツリー内の差分ファイルを表示
    git diff	ファイル内の差分箇所を表示
    git mv	ファイルを移動／ファイル名を変更
    git stash	作業ツリーの状態を一時的に保存
    git branch	ブランチの作成／一覧表示
    git checkout	処理対象ブランチの切り替え
    git merge	別のブランチから変更点をマージ
    git rebase	派生元ブランチに変更点をマージ

関数とラムダ式の利用

    
        fun f(x: Int):Int{ return x * x + 2 * x + 1}
        と同じものをラムダ式で定義することもできる↓
        var f = {x: Int -> x * x + 2 * x + 3 }
        型を指定するラムダ式もある↓
        var f: (Int) -> Int = {x: Int -> x * x + 2 * x + 3 }
    
    
質問の仕方

    自分で調べて、自分で考えて、調べたことや考えたことを根拠に記述をしてそれを見せて
    「こう考えました」と質問するのが良い方法です。
    計算過程など消しゴムで消して「証拠隠滅」をしないようにしましょう。
    自分の間違いをさらけ出せる人はよく伸びる人です。
    また自分で考えたことや自分が導いた結論を完結に言うことも大切です。
    

    
改善の余地がある質問

    全部わかりません。　→　具体的にわからないところを絞る
    何もわかりません。　→　問題文が理解できたかどうかを伝える。
    解説のここがわかりません。　→　自分ではどう考えたのか。


    
できる人の質問の仕方と傾向

    自分で調べて、自分で考えて、調べたことや考えたことを根拠に記述をしたものを見せて
    「こう考えました。正しいですか。」と質問する。

    簡潔に質問したいことを口頭で述べる。

    質問したい問題を簡潔に伝わるように説明できる。

    質問したいことをすぐに取り出せる。

    短時間で大量の解決をする。

    質問が多い。

    難しい質問よりも基本を正しく深く理解しようとする質問が多い。

    理解するのに時間がかかりそうなことは考え方をメモして再考する。

    感情的にならず論理的に考える。（なんとなく不安というのを根拠にしない）

    根拠を一つだけに限定せずに多面的に捉えようとする。

    質問の出発点が明確。

    記憶よりも記録を元に質問する。

    
学習が上手な人の特徴

    わからなければとりあえず試行錯誤する。
    手を動かしながら考える。
    頭の中を整理するために書き出す。
    情報を整理するために書き出す。
    迷う暇があれば書いて考える。
    考えたものを見せてから質問する。
    結論を先に話す。


    
    
理系科目の解答解説との向き合い方

    理系科目の解答解説は計算手順を省略してあることが多いです。
    計算がわからない場合に自分で手を動かさずに解答解説だけを見てわからないことはよくあります。
    自分で一つ一つ行を追って「手を動かす」ことが重要です。
    そうして計算力がついたら見るだけでも理解できる部分がかなり増えます。
    まだ見るだけで理解できるレベルに達していない状態で解答解説を見るだけの学習方法は詰まりやすくなります。


    
学習レベル診断

    問題中に自分で説明できない用語が出てきても気にしない。
    解くだけで答え合わせをすぐにしない。
    解いた直後に答え合わせをする。
    解答解説を見てもわからないことがある。
    解答解説を見て何がわからないかがわからない。
    解答解説を見てわかったと思うが類題は解けない。
    解答解説を見てわからないことがほとんどない。


    
成績が上がりにくい人の特徴

    わからないときに他の本やネットなどいろんなものに手を出す。
    わかるように自分のレベルを上げる方向で考えるのが良い。

    いろんな問題集に手を出して一つのものを完璧にしようとしない。

    言うことを聞かない。
    素直じゃない。

    片付けができない。
    問題情報の整理整頓ができない。

    質問の目的が不安を解消するためであって学力を上げるためではない。
    不安を解消するための勉強を優先してしまって学力を上げることを最優先にしていない。

    計算に慣れていないから思考部分に集中できない。

    公式の意味を理解せずにただ使っているだけ。

    すぐに諦める。考えない。まあいいや、となる。

    根拠が論理的でない。感情の方を優先させる。
    どんなに正しいことでも受け入れられずに自分の間違った感覚を優先させてしまう。


    
保護者への提言

    大学入試レベルは上がっています。
    以前は難しい問題とされていたものが現在では頻出の応用問題という扱いになることもあります。
    より早く、より高いレベルに到達するための工夫がここ数年でも飛躍的に進歩しています。
    科目によって特徴は異なります。ある科目には適した学習方法が他の科目では効率の悪い学習方法となることもあります。
    学習者、保護者、指導者がそれぞれに異なる学習方法が良いと考えることが学習者の学力向上にブレーキをかけます。
    最も良いと思われる学習方法はこちらから提言しますが、もしも提言内容と異なる行動が見られる場合には学習方法についての話し合いが必要です。
    気持ちのブレーキは学力向上にとって非常に大きなブレーキとなります。
    学習者自身が納得していない学習方法があるのならばベストな方法よりもベターな方法を取った方が良いこともあります。
