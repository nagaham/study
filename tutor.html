講師を選ぶときの注意点

    担当講師がどこまでできるのかを確認しましょう。
    よくある失敗が文系の講師に理系科目を習っていて大学入試レベルの対応ができなくなり交代というものです。
    理系を目指す場合は最終的に数学を全範囲することになります。
    特に数学に多いのですが数1A2Bまでを教えて数3になると他の講師に依頼する者がいます。
    理系にとっては数3を前提に1A2Bを学習しなければいけません。それがわからない者が理系志望の生徒を教えるのは損失です。
    自分では問題が解けないのにも関わらず講師を名乗るものが世の中には多く存在します。
    実は指導ができないけれども指導をしている風のニセ講師を見抜くには授業時の態度を見ればわかります。
    常に解答解説を片手に解説頼りの説明ばかりをしている場合は自分では解けない可能性があります。
    適当に教科書レベルの問題などを質問して本当に解けるのかどうかを確認してみましょう。

    どの科目であっても少なくともその大学の受験を代わりにできるくらいのレベルの講師に依頼するべきです。
    世の中の講師の半分以上はそれができていません。解答解説を見ながらそれっぽいことを言っているので注意しましょう。


    
高校情報の講師に必要なスキル


    タイピング能力
    ほとんどキーボードを見ずに英数字や記号についても入力することができる。

    言語(指導範囲内のレベル)
    html css javascript python scratch

    vcs
    git と github を用いてリモートリポジトリを public な環境にデプロイすることができる。

    実装能力
    数学や理科などの理系科目の内容をプログラミングの課題に落とし込んで実装することができる。

    開発環境構築
    Mac Windows において必要な環境を構築することができ、オフラインでも実行可能な環境を用意できる。

    自走能力
    上記の内容について問題が起きた時に自力で解決できる。
